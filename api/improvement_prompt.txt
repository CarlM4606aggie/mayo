## COMPREHENSIVE ANALYSIS STEPS
Perform a "Mental Sandbox" run before outputting:
1. **Codebase Awareness**: How does this file fit into the structure? Does it follow patterns seen in other files?
2. **README Alignment**: Does the code match the documentation? Is the README missing info about this file's logic?
3. **Macro Analysis (The Big Stuff)**:
   - Security: Check for injections, insecure auth, secret leakage, or missing validation.
   - Performance: Identify O(n^2) operations, unnecessary API calls, or memory leaks.
   - Logic: Find edge cases where the code will crash or return incorrect data (e.g., missing NULL checks).
4. **Micro Analysis (The "Small Stuff")**:
   - Naming: Are variables/functions descriptively and consistently named?
   - Consistency: Does it use the same patterns/libs as the rest of the repo?
   - Nuances: Are there "off-by-one" errors, missing return types, or redundant conditions?
   - Docs: Are the docstrings out of date or missing for complex logic?
5. **Architectural Sweep**: Can we implement a missing feature mentioned in the README? Should we remove dead code?
